This action maps values found in defined areas of extracted PDF text to specific keys.&#x20;

**Obs:** The action requires configuration to define areas, keys, and filters. It is designed to work with the output of the **Pdf Extract Text** action.

![](/images/docs/placeholder-areamapper1.png)

## Inputs & Outputs

- **\[ I ] Extracted Words:** Collection of words extracted from a PDF using the **Pdf Extract Text** action.
- **\[ I ] Configuration:** JSON configuration that defines areas, keys, filters, and rules for mapping.
  - Options include page filters, row limits, regex filters (case-sensitive/insensitive), element concatenation, and top/bottom/left/right bounds.
- **\[ O ] Mapped Areas:** Key-Value list containing the mapped values for each configured area.

## Example Usage

- Drag **Area Mapper** onto the canvas.
- Declare variables:
  - `extractedWords` → Output of **Pdf Extract Text** action
  - `areaConfig` → JSON with mapping configuration
- Example `Configuration` JSON:

```json
[
  {
    "Key": "InvoiceNumber",
    "Pages": [1],
    "TopLimit": null,
    "BottomLimit": null,
    "LeftLimit": null,
    "RightLimit": null,
    "CaseSensitiveRegex": "[0-9]{8}",
    "MaxNoOfRowsPerTarget": 1,
    "TakeFirstElements": 1
  },
  {
    "Key": "CustomerName",
    "Pages": [1],
    "CaseInsensitiveRegex": "name",
    "TakeLastElements": 1
  }
]
```



## Configuration Options Reference

Use the `Configuration` JSON to define what to extract and how. Each entry describes one **area** (and its key) to map.

| Option                 | Type      | Description                                                                     | Default / Notes                                        |
| ---------------------- | --------- | ------------------------------------------------------------------------------- | ------------------------------------------------------ |
| `Key`                  | string    | The name under which matched values will be returned.                           | **Required**                                           |
| `Pages`                | number\[] | Pages to scan (1-based). If omitted/empty, scans **all pages**.                 | Defaults to all pages                                  |
| `TopLimit`             | number    | Y coordinate of the area’s **top** boundary.                                    | If null, inferred via `TopWordLimit` or page top       |
| `BottomLimit`          | number    | Y coordinate of the area’s **bottom** boundary.                                 | If null, inferred via `BottomWordLimit` or page bottom |
| `LeftLimit`            | number    | X coordinate of the area’s **left** boundary.                                   | If null, inferred via `LeftWordLimit` or leftmost      |
| `RightLimit`           | number    | X coordinate of the area’s **right** boundary.                                  | If null, inferred via `RightWordLimit` or rightmost    |
| `TopWordLimit`         | string    | Word used to infer `TopLimit` (uses that word’s **Ybottom**).                   | Optional                                               |
| `BottomWordLimit`      | string    | Word used to infer `BottomLimit` (uses that word’s **Ytop**).                   | Optional                                               |
| `LeftWordLimit`        | string    | Word used to infer `LeftLimit` (uses that word’s **Xright**).                   | Optional                                               |
| `RightWordLimit`       | string    | Word used to infer `RightLimit` (uses that word’s **Xleft**).                   | Optional                                               |
| `CaseSensitiveRegex`   | string    | Regex filter applied to row strings (case **sensitive**). Keeps only matches.   | Optional                                               |
| `CaseInsensitiveRegex` | string    | Regex filter applied to row strings (case **insensitive**). Keeps only matches. | Optional                                               |
| `MaxNoOfRowsPerTarget` | number    | If `> 1`, concatenates rows in groups of this size (newline-separated).         | Default `null`/`1` = no grouping                       |
| `TakeFirstElements`    | number    | Keep only the first **N** matches after prior steps.                            | Default `0` (no limit)                                 |
| `TakeLastElements`     | number    | Keep only the last **N** matches after prior steps.                             | Default `0` (no limit)                                 |

### How areas & rows are computed

- Words are selected if their positions fall **inside** the rectangle:
  `Left < Xleft` and `Xright < Right` and `Ytop < Top` and `Ybottom > Bottom`.
- Selected words are grouped by `PageRow`. Each row’s words are ordered by `PageRowWordOrder`, then joined with spaces to form a row string.
- If `MaxNoOfRowsPerTarget > 1`, rows are concatenated in chunks of that size, separated by newline.
- `TakeFirstElements` / `TakeLastElements` are applied after grouping.
- `CaseSensitiveRegex` and/or `CaseInsensitiveRegex` keep only rows that match.

### Minimal Example

```json
[
  {
    "Key": "InvoiceNumber",
    "Pages": [1],
    "RightWordLimit": "Invoice",
    "CaseSensitiveRegex": "\\b\\d{8}\\b",
    "TakeFirstElements": 1
  },
  {
    "Key": "LineItems",
    "Pages": [1, 2],
    "LeftLimit": 80,
    "RightLimit": 520,
    "TopWordLimit": "Items",
    "BottomWordLimit": "Total",
    "MaxNoOfRowsPerTarget": 2,
    "CaseInsensitiveRegex": "^[A-Z].+\\d+(\\.\\d{2})?$"
  }
]
```
